这份方案旨在将 **LLM（大语言模型）** 的生成能力与 **Android 原生性能** 深度结合，打造一款既具备高审美价值，又能解决初学者“学了不会用”痛点的英语学习工具。

# 🚀 AI 驱动型英语学习 App (代号：Juno) 完整方案

## 1. 产品核心定位

* **愿景：** 让每一位英语初学者拥有一位 24 小时在线、懂用户兴趣、能即时纠错的私人助教。
* **核心竞争力：** 拒绝死板的词库，实现“内容输入 -> 智能解析 -> 场景化复习”的闭环。
* **直击痛点：打破“孤岛效应”(The Island Effect)**
  * 单词背诵不应是点状映射（如单一的“中文-英文”），而必须融入网状语境。脱离语境会导致用户能够认出 `Apple`，却无法用其描述口感（如 `crisp/mealy`）。本方案的所有机制均服务于建立**“场景-英文”**的强神经连接，彻底激活所谓的“死词汇”。

## 2. 功能模块详细设计
### 基础功能：背单词app,例如市场上的百词斩，不背单词。

### 核心功能零：科学记忆引擎 (SRS & Chunking)
* **功能逻辑：** 引入基于艾宾浩斯遗忘曲线的**间隔重复系统 (SRS)**，追踪每个单词/词块的记忆留存率。
* **教学理念：** 强调**语块 (Chunks)** 学习，而非孤立背词。系统自动识别短语搭配，作为独立记忆单元。

### 核心功能一：LLM 场景化叙事 (Story-Driven Learning)

* **功能逻辑：** 系统根据用户当日需复习的单词（如 5 个），生成一段长度约 100 词的短文。
* **差异化：**
* **难度自适应：** LLM 严格限制在 **CEFR A1/A2** 级别语法。
* **角色带入：** 用户可选择故事风格（职场、校园、科幻、日常生活）。


* **执行点：** 使用 Prompt Engineering 确保单词在文中被**加粗显示**并具备**正确语境**。

### 核心功能二：小红书式科普卡片 (Aesthetic Knowledge Cards)

* **功能逻辑：** 将枯燥的词典解释转化为精美的卡片。包含：单词、音标、LLM 生成的“地道俗语”、一个冷知识或助记梗。
* **审美设计：** * 支持 **一键换肤**（莫兰迪色系、极简白、深邃黑）。
* 支持 **导出分享**，利用 Android 的 `Canvas` 绘制长图，方便用户打卡到社交平台。



### 核心功能三：全方位感知输入 (Multimodal Input)

* **链接提取 (Link-to-Study)：**
* 用户分享 YouTube/网页链接，App 通过后台解析文本或字幕。
* **LLM 自动提炼：** 自动识别文中的重点词汇、固定搭配和语法点。


* **系统级 OCR 悬浮窗：**
* 通过 `AccessibilityService` (辅助功能) 或 `MediaProjection` 实现屏幕文字抓取。
* **快捷入口：** 在其他 App（如 Twitter, Chrome）中一键呼出，即刻查询并收藏。



### 核心功能四：AI 闭环听评测 (Smart Dictation & Feedback)

* **模仿真人听写：** 调用 **OpenAI TTS (HD)** 或 **Azure Neural Voice**，模拟略带呼吸声的真人语速， 也支持声音克隆，即可以用自己的声音，也可以用熟人的，降低陌生感，增加亲切感真实感。
* **实时纠音：**
* 用户录音后，通过 **Whisper API** 转文字，与标准答案对比。
* **LLM 诊断：** 不仅给出分数，还指出具体发音错误（如：“你的 /θ/ 发音太像 /s/ 了”）。
* **Shadowing (跟读模仿)：** 提供原音与用户录音的波形对比，支持分段变速循环，攻克连读、弱读等语音现象。

### 核心功能五：AI 追问式外教 (Proactive Probing & Multi-Tutor)

* **功能逻辑：** 拒绝“你读我评”的被动式单向反馈，系统底层转向**主动对话式引导**。
* **应用场景：** 当用户刚背完/复习完 `Adventure` 时，AI 会主动发问：“如果你现在有 100 万，你想去哪里冒险？”强迫并在鼓励下让用户调用“活跃词汇”进行表达。
* **多外教陪伴模式：** 支持切换多种性格与口音的 AI Tutor（例如：严谨的大学讲师、热情的旅行博主等），创造更真实的跨文化语境和多样化的输入。

### 核心功能六：实景视觉锚定 (Visual Reality Anchoring)

* **功能逻辑：** 利用计算机视觉与 LLM + 图像生成（考虑整合 Nano Banana 2 等相关技术方案），将单词死死锚定在用户的真实生活场景中。
* **执行方案：**
  * 用户拍摄一张自己的书桌、办公室或卧室真实照片。
  * 系统自动识别图中物体（如：Monitor, Stapler, Succulent），并围绕这些专属且私密的物品片段编写一个连贯由于英语冷知识。
* **降维打击：** 相比于死记硬背课本上陌生的“桌子”，记住属于自己的那张书桌所形成的神经突触连接要深厚得多这不仅是背单词，更是在给用户的真实生活空间**“打上英文标签”**。

### 核心功能七：缓存模块

## 3. 技术架构方案 (最新全栈栈)

| 层次 | 技术选型 | 备注 |
| --- | --- | --- |
| **前端 UI** | **Jetpack Compose** | 实现丝滑动效、响应式布局及离屏渲染导出长图(`GraphicsLayer`)。 |
| **本地存储** | **Room DB + DataStore** | 存储单词本、学习进度、本地缓存。 |
| **云端/后端** | **BFF (Node.js/Go) 或 Firebase/Supabase** | **安全关键**：绝对不在客户端暴露 LLM API Key，通过 BFF 代理请求并实现云端数据同步。 |
| **AI 模型接入** | **OpenAI / DeepSeek / Gemini** | 后端通过长连接或 SSE (Server-Sent Events) 流式返回数据至客户端。 |
| **端侧 AI/CV** | **Gemini Nano (AICore) + Google ML Kit** | Nano 用于离线文本纠错；ML Kit 提供极速的端侧 OCR 文字识别。 |
| **后台任务** | **WorkManager** | 处理每周预约提醒和链接异步解析。 |
| **多媒体框架** | **AndroidX Media3 (ExoPlayer) + Oboe** | 确保低延迟的音频播放与录制，支持最新的媒体格式。 |

## 4. UI/UX 设计原则

> **"智能而不沉重，美观而不繁琐。"**

* **卡片化交互：** 采用类似 Tinder 的卡片滑动操作进行单词复习，减少心理压力。
* **微动效：** 在点击收藏、完成听写时加入触觉反馈（Haptic Feedback）和轻微的缩放动画。
* **沉浸式陪伴：** “集中陪伴模式”开启时，界面自动进入极简状态，仅保留 AI 助教的语音波形动画。

## 5. 全功能演进路线图 (Comprehensive Roadmap)

考虑到功能的复杂度升级，我们将原来的三个阶段延展为四个精心编排的开发冲刺期：

### 第一阶段：基座搭建与记忆引擎 (The MVP - 5周)
* **基础建设**：搭建 Jetpack Compose UI 框架与 BFF 后端安全代理服务。
* **完成[基础功能]与[核心功能 0]**：实现极简的看词背词功能，并打好本地 **SRS (间隔重复系统)** 数据库底座。
* **完成[核心功能 1]**：全面接入 LLM，完成 **从生词列表智能拼装场景化短文故事** 的闭环开发。

### 第二阶段：全场景捕获与语音测评突破 (6周)
* **完成[核心功能 3]**：研发外部输入系统，包含 **底层 OCR 悬浮窗取词** 技术与 **网页链接分享拉取 (Link-to-Study)** 的智能过滤机制。
* **完成[核心功能 4]**：全面集成 Media3 以及音频 AI (TTS & Whisper)，上线带有波形逐帧对比的 **听写评分与 Shadowing 跟读模块**。

### 第三阶段：AI 强无敌交互与“破岛行动” (6周)
* **完成[核心功能 5]**：推出 **AI 追问式多外教 (Proactive Probing)**。引入不同性格的 Agent Prompt 设置，AI 能够主动引发日常化场景对话。
* **完成[核心功能 6]**：研发爆款武器 **实景视觉锚定 (Visual Reality Anchoring)**，集成视觉识别大模型，真正让英语“长在用户家里的书桌上”。

### 第四阶段：高潮审美与传播体系 (3周)
* **完成[核心功能 2]**：设计打磨 **小红书式高审美科普卡片**。接入 `GraphicsLayer` 长图导出引擎，并提供多套 UI 主题供用户展现品味。
* **UX/体验闭环**：全局丝滑动效验收（如类似 Tinder 的卡片滑动、微小的体感震动反馈），完成安静私密的极简陪伴模式定时器逻辑。

* **版权规避：** 链接解析仅抓取文本用于个人学习，需在用户协议中明确。
